# MIPS Prefix XOR Simulator

A Python-based MIPS processor simulator that executes a prefix XOR algorithm. The assembly program computes the running XOR of an integer array `A` and stores the results in array `P`, simulated cycle-by-cycle through a 5-stage pipeline.

---

## What It Does

Given array `A = [5, 2, 7, 3, 1, 4, 6, 8, 9, 10]`, the program computes:

```
P[0] = A[0]
P[1] = A[0] ^ A[1]
P[2] = A[0] ^ A[1] ^ A[2]
...and so on
```

The result is stored in array `P` in data memory, and verified at the end of the simulation.

---

## Project Files

| File | Description |
|---|---|
| `AssemblyCode.txt` | The original MIPS assembly source for the prefix XOR algorithm |
| `instructionmemory.txt` | Binary machine code (32-bit instructions), generated by the MARS assembler |
| `datamemory.txt` | Binary data memory initialised with array `A`, generated by the MARS assembler |
| `processor.py` | Python MIPS processor simulator — fetches, decodes, and executes instructions |

> **Note:** `instructionmemory.txt` and `datamemory.txt` were assembled and exported using the **MARS MIPS Assembler and Runtime Simulator**. Each line is a 32-bit binary word representing an instruction or data value.

## Prerequisites

- **Python 3.7+** — no third-party libraries required, only the Python standard library.
- The four project files must all be in the **same directory** before running.

You can verify your Python version with:
```bash
python --version
```

## How to Run

1. Clone or download all four project files into the same folder.
2. Open a terminal in that folder.
3. Run the simulator:

```bash
python processor.py
```

The simulator will print a cycle-by-cycle trace of all 5 pipeline stages (IF, ID, EX, MEM, WB), followed by a register dump, the final data memory contents, and a verification table confirming each `P[i]` value is correct.

## Core Concepts

### Prefix XOR
The prefix XOR of an array is a sequence where each element is the cumulative XOR of all preceding elements including itself. It is commonly used in range XOR queries, since `A[l] ^ ... ^ A[r]` can be computed in O(1) using `P[r] ^ P[l-1]`. The MIPS program implements this iteratively with a loop from index 1 to 9.

### 5-Stage Pipeline Simulation
`processor.py` models the classic MIPS pipeline — each instruction passes through five stages every cycle:

- **IF (Instruction Fetch)** — reads the 32-bit binary instruction from `instructionmemory.txt` at the current PC.
- **ID (Instruction Decode)** — decodes the opcode, function code, register indices, and immediate fields; reads source register values.
- **EX (Execute)** — performs the ALU operation (add, XOR, shift left logical), resolves branch conditions, and computes memory addresses.
- **MEM (Memory Access)** — reads from or writes to `datamemory.txt` for `lw` and `sw` instructions; the file is updated in real time on every store.
- **WB (Write Back)** — writes the computed result back to the destination register.

### MIPS Instructions Used

| Instruction | Purpose |
|---|---|
| `la` (lui + ori) | Load the base address of arrays `A` and `P` |
| `lw` / `sw` | Read and write 32-bit words to data memory |
| `sll` | Multiply the loop index by 4 to get a byte offset |
| `add` / `addi` | Pointer arithmetic and loop counter increment |
| `xor` | Compute the running XOR between consecutive elements |
| `beq` | Branch to exit once all 10 elements are processed |
| `j` | Jump back to the top of the loop |
| `syscall` | Halt the processor (exit code 10) |

### Memory Layout

The simulator uses fixed base addresses matching the MARS assembler defaults:

- **Text segment:** `0x00400000` — holds instructions from `instructionmemory.txt`
- **Data segment:** `0x10010000` — holds array `A` (words 0–9) and array `P` (words 10–19) from `datamemory.txt`
